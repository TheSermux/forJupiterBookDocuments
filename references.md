# Принципы функционирования справочника документации

Справочник документации обновляется про принципу клонирования единого справочного репозитория на рабочий сервер командой

```git clone https://github.com/TheSermux/forJupiterBookDocuments.git```

После загрузки запускается виртуальное окружение jbVenv

```shell
source jbVenv/bin/activate
```

Сам справочник генерируется на основе репозитория и помещается в репозиторий веб-сервера Apache, развернутого на рабочей виртуальной машине:
```shell
sudo mv /var/www/html /var/www/html_backup # Бэкап имеющегося справочника
```

```shell
jupyter book build --html # генерация нового справочника (производится внутри папки склонированного репозитория)
```
```shell
sudo cp -r _build/html /var/www/html # копирования нового справочника на прод
```
